Reflection On Week 1

Written on the 22nd of February 2019 by Maddie

What Worked

  • Finally started the Language Learning Application project (again);
  • The work for this project so far has been consistent and motivating;
  • Wrote some project documentation and gained a pretty clear idea of what I want to build;
  • Built the LLA3 API and currently it is in a working condition (good enough for my future Angular client);
  • Had the pleasant experience of working again with Node, Express and MongoDB;
  • Testing the API using Postman - such a useful tool!

What Didn't Work

  • Stressed for time - it has been a struggle for find enough time to work;
  • Unsure of some of the code that I wrote - either because it could be written better or because there might be a better way to make it do the same thing;
  • My Git commits - should have committed more often so there would not be such huge differences between the versions;
  • API Error - sending invalid JSON through the HTTP request triggers a 500 error so I need to fix that;
  • Lack of knowledge - I discovered that I need to refresh my knowledge about certain topics.

Current Issues

  • The variable type of my dependencies in app.js and in the other api files - Deciding between let, var and const.
  • Should I use babel.js - ECMAScript 5 vs ECMAScript 6 Browser Compatibility Concerns
  • API Future Improvements
    • API Documentation would be nice;
    • Based on the current app-tree diagram I might need to create one or two more endpoints to retrieve the exact data that I want to display on the frontend, for example all the languages / categories for a language for all the words in the database;
    • Validation and sanitization for the incoming HTTP requests, for example checking that the data I receive is in a correct JSON format;

Possible Solutions

  • Plan and manage your time better!
  • Look up some developer best practices and stick to a code standard!
  • Be more mindful of your commits!
  • Do more testing!
  • Don't forget to document everything because you WILL forget!

Next Week's Plan

  • Start work on the Angular client (yay!);
  • Try to add the necessary improvements to the API;
  • Friday - write the next work log entry!
